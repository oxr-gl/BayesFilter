# Whole-Block Integration Audit: Expanded High-Dimensional Block After Deep Rewrites

**Date:** 2026-06-15  
**Scope:** `docs/main.tex`, `ch33`, and the expanded `ch34`--`ch38` sequence after the deep rewrites of `new ch35`, `new ch36`, and the strengthening of `new ch37`

## Overall assessment

The expanded high-dimensional block is now past the point of mere architecture
activation.  It is a real monograph sequence with distinct chapter roles and a
stable build.  The strongest change since the first post-migration audit is that
both method chapters now have a more credible internal job:
- `new ch35` is now much more clearly the forward sparse-grid / fixed-cloud
  method chapter,
- `new ch36` is now much more clearly the retained-object / TT / KR chapter,
- and `new ch37` is no longer only a thin bridge but a recognizably shared
  fixed-branch same-scalar chapter.

The block is therefore now in a different state: the next priority is **no longer
another obvious single deep rewrite target**.  The highest-value next step is now
whole-block normalization: labels, chapter-local voice, compatibility residue,
and import/export wording cleanup across the entire sequence.

## Strongest remaining integration issues

### 1. Block-wide normalization residue is now the dominant issue
The largest remaining problem is not that one chapter is badly wrong. It is that
several chapters still visibly carry the residue of the migration path:
- inherited p-number label namespaces,
- compatibility alias labels,
- older role names,
- seeded section titles that do not yet fully match the new architecture.

This is now more important than any single local rewrite, because the block’s
conceptual architecture is mostly in place.

### 2. `ch34` -> `ch35` handoff can still be made more explicit
The handoff is now structurally correct, but still slightly under-articulated in
a formal import/export sense.  `new ch34` conceptually prepares rule-family
foundations, and `new ch35` properly specializes them, but the explicit chapter
contract language could still be tightened so the reader sees the handoff as a
deliberate theorem-to-method transition rather than just chapter order.

### 3. Transitional compatibility labels are still doing real work
The block still compiles partly because the new chapters provide compatibility
labels for the old compressed architecture.  That was the right short-term move,
but it is now the clearest signal that the architecture has not yet been fully
renormalized.

### 4. The final p47/p50 explanatory weight is better but still not complete
- `new ch35` now carries the sparse-grid method more clearly.
- `new ch36` now carries the retained-object method more clearly.
But there is still room to deepen explanatory continuity once the normalization
pass settles the architecture.

## Should the next priority be another deep rewrite or whole-block normalization?

### Recommendation: whole-block normalization now has the highest value.

Reason:
- There is no longer an obvious single chapter whose role is still fundamentally
  wrong.
- The recent deep rewrites corrected the biggest role/fidelity mismatches in
  `new ch35` and `new ch36`.
- The shared fixed-branch chapter is now substantial enough to function.
- The strongest remaining defects are cross-cutting and architectural in a lighter
  sense: labels, titles, import/export wording, compatibility residue, and final
  monograph voice alignment.

That means the next pass should likely focus on:
1. label normalization strategy,
2. compatibility-label cleanup plan,
3. chapter-local voice normalization,
4. handoff wording and export packaging,
5. a final read-through of the `ch33` + `new ch34`--`new ch38` sequence as one
   narrative.

## If one chapter still merits special attention

If forced to nominate one chapter for one more source-heavy pass, it would still
be:
- `new ch36`

But the reason is no longer structural failure. It would now be only because p50
is the denser source manuscript and the retained-object lane still has the most
residual survey-like material.

Still, I do **not** think that is the highest-value next move. I think the block
has crossed the threshold where a broad normalization pass will likely produce
more value than another immediate deep-local rewrite.

## Recommended next-step order

1. **Whole-block normalization pass**
   - labels,
   - chapter-local voice,
   - compatibility residue,
   - import/export wording,
   - title/role consistency.
2. Rebuild and reread the full expanded block as one narrative.
3. Only then decide whether `new ch36` still needs another source-heavy deepening
   pass.

## Conclusion

The expanded block is no longer in “architectural rescue” mode and no longer in
obvious “single chapter still badly wrong” mode.  The best next step is a
whole-block normalization pass.  If that pass still reveals one chapter that
visibly under-carries its source manuscript, `new ch36` is the most likely
candidate for one further deepening round.
