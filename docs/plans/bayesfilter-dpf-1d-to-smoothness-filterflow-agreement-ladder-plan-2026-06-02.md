# Plan: 1D-to-Smoothness Filterflow Agreement Ladder

## Scope

This plan creates and executes a reviewed agreement ladder that moves from the
matched 1D scalar-state LGSSM audit toward the local executable filterflow
smoothness LGSSM one axis at a time. The purpose is to identify the first
actual BayesFilter-vs-filterflow discrepancy, if any. It is not a correctness
proof for either implementation.

Lane boundary: BayesFilter-owned DPF implementation/evidence lane only. Do not
edit production `bayesfilter/`, `tests/`, monograph chapters under
`docs/chapters/`, high-dimensional nonlinear filtering lane artifacts,
student/vendored code, DSGE/NAWM artifacts, or `.localsource/filterflow`
source. Keep BayesFilter-owned implementation under
`experiments/dpf_implementation/tf_tfp/`; keep reports under
`experiments/dpf_implementation/reports/` and `docs/plans/`.

## Evidence Contract

Primary question: at which one-axis transition, if any, does BayesFilter first
diverge from the local executable filterflow reference?

Primary comparator: current local patched executable filterflow checkout under
`.localsource/filterflow`, identified by HEAD commit plus local diff/status
fingerprint. The branch string is descriptive only.

Primary pass criterion per rung: BayesFilter/filterflow trigger flags,
promoted scalar value, forward ledger, transport matrix, post-transport
particles, log weights, and residual values agree within predeclared comparison
tolerances for the exact cells executed in that rung. Gradient/FD quantities
are diagnostic only until a rung explicitly marks the scalar/gradient contract
as reconciled.

Veto diagnostics:

- filterflow subprocess blocker;
- nonfinite values;
- trigger mismatch;
- scalar/ledger mismatch;
- transport-matrix mismatch;
- residual mismatch;
- CPU-only manifest failure;
- comparator drift;
- forbidden path/import use.

Diagnostic-only quantities:

- absolute transport residual magnitude when shared by both implementations;
- filtered-state RMSE;
- runtime;
- AD/FD gradients unless scalar/gradient contract is explicitly reconciled.

Required governance correction:

- shared high residual is not a BayesFilter-vs-filterflow failure;
- a residual gate can veto only if BayesFilter and filterflow residuals differ
  beyond comparison tolerance, or if nonfinite values occur;
- if both implementations produce the same questionable residual, record
  `shared_quality_diagnostic`.

## Rungs

Run rungs sequentially. Stop only on an actual implementation discrepancy or a
hard blocker. If a rung passes agreement with shared quality concerns, continue.
Within each rung, execute cells in the deterministic order listed below and
record the first failing cell by `(rung_id, cell_index, cell_id)`. A rung with a
grid or surface stops at the first mismatch in that ordered grid after writing
all diagnostics for the failing cell; it does not continue to later cells as if
the first-discrepancy objective were a sweep objective.

1. `R1_1d_T100_filterflow_observation_path`:
   keep scalar-state model, matched annealed transport, fixed initial particles
   from the existing controlled 1D audit, deterministic transition-noise ledger,
   `theta=0.7`, `Q=0.04`, `R=0.04`, `N=4`, `epsilon=0.25`,
   `scaling=0.9`, `convergence_threshold=1e-6`, and `max_iterations=200`;
   switch only observations to a fixed filterflow-generated path of length
   `T=100` generated by the local executable filterflow reference with
   `data_seed=123`. Cell order: the single `T100_filterflow_observation_path`
   cell.

2. `R2_1d_initial_particle_generation`:
   additionally replace the fixed initial particles with the filterflow
   executable initial-particle draw for the same scalar model. The draw must be
   captured from a filterflow subprocess, stored in the JSON artifact, and then
   fed as an explicit tensor to BayesFilter. For the smoothness script,
   initial particles are drawn from the NumPy `data_seed=123` random state
   after data generation; `filter_seed=1234` governs SMC seed splitting, not
   initial-particle generation. Cell order: the single
   `filterflow_initial_particles_data_seed_123_after_data_generation` cell. If the scalar-state
   filterflow subprocess cannot expose those particles without mutating
   `.localsource/filterflow`, record a structured blocker.

3. `R3_1d_transition_random_stream`:
   additionally replace the deterministic transition-noise ledger with the
   filterflow executable transition/proposal random stream for the same scalar
   model, captured by subprocess and replayed explicitly in BayesFilter. Cell
   order: the single `filterflow_transition_stream_seed_1234` cell. If the
   stream cannot be captured without mutating `.localsource/filterflow`, record
   a structured blocker; do not invent a near-match.

4. `R4_resampling_policy_trigger_semantics`:
   additionally replace the inherited threshold check with the executable
   filterflow `NeffCriterion(0.9999, True)` semantics. Promoted comparison:
   pre-resampling ESS, trigger flag, and trigger-time state agree at every time
   step. Cell order: the single `neff_09999_true_trigger_semantics` cell.

5. `R5_scalar_contract`:
   promoted scalar is exactly the executable filterflow SMC scalar:
   `reduce_mean(final_state.log_likelihoods)` over batch, where each
   per-trajectory log likelihood is the sum over time of
   `reduce_logsumexp(unnormalized_log_weights_t, axis=particles)`.
   Sign is positive log likelihood, not negative log likelihood. Correction
   terms are included only if the executable filterflow state includes them in
   `log_weights` before that `reduce_logsumexp`. Gradients remain diagnostic.
   Cell order: `total_log_likelihood`, then `batch_mean_log_likelihood` if
   `batch_size > 1` is introduced by a later rung.

6. `R6_theta_grid_surface`:
   move from a single theta to an ordered 1D scalar theta grid. Bounded
   execution uses theta values `[0.5, 0.7, 0.9]` first; if these pass, the
   runner records a structured diagnostic-only comparison against the
   filterflow smoothness `mesh_size=20` default but does not promote that
   full mesh unless runtime remains bounded. Cell order is ascending theta.

7. `R7_2d_constant_velocity_state`:
   move from scalar state to the 2D constant-velocity model:
   `A(theta)=diag(theta_1, theta_2)+[[0,1],[0,0]]`,
   transition covariance `[[1/3,1/2],[1/2,1]]`, observation matrix `[[1,0]]`,
   observation covariance `[[0.01]]`.

   Use the smoothness defaults `T=100`, `batch_size=1`, `n_particles=50`,
   `epsilon=0.25`, `scaling=0.85`, `convergence_threshold=1e-6`,
   `max_iter=500`, `resampling_neff=0.9999`, `data_seed=123`,
   `filter_seed=1234`, and `optimal_proposal=True`. Cell order: one
   2D constant-velocity cell at transition parameters `(theta_1, theta_2)` set
   to the smoothness script's center mesh point nearest `(1.0, 1.0)`.

8. `R8_gradient_smoothness_surface`:
   compare the executable filterflow gradient object
   `GradientTape.gradient(reduce_mean(final_state.log_likelihoods), transition_matrix)`
   against BayesFilter `GradientTape` for the same scalar and fixed random
   objects. Also record finite differences using filterflow default
   `diff_epsilon=1e-2`. Promotion is allowed only for scalar equality and
   gradient object identity/shape/normalization; numerical AD/FD agreement is
   diagnostic unless the result explicitly sets
   `scalar_gradient_contract_reconciled=true`. Cell order: the same center
   cell as R7, then a bounded `[0.5, 1.0, 1.5]` diagonal perturbation grid if
   the center cell passes.

- default `data_seed=123`;
- default `filter_seed=1234`;
- default `optimal_proposal=True`;
- transition covariance uses executable `I_2` convention in prior LGSSM table
  reproductions; the 2D smoothness model uses the explicit covariance above.

Authoritative-for-ladder defaults: `T`, `batch_size`, `n_particles`,
`epsilon`, `scaling`, `convergence_threshold`, `max_iter`,
`resampling_neff`, `data_seed`, `filter_seed`, `optimal_proposal`, model
matrices/covariances, and the promoted scalar formula in R5. Diagnostic-only
or bounded substitutes: `mesh_size=20` is recorded but the full mesh is not a
promotion requirement unless all bounded cells pass within runtime; `savefig`
is disabled in wrappers; `use_xla=False` is retained; `diff_epsilon=1e-2` is
used only for FD diagnostics.

## Stop Conditions

- exact Claude command unavailable;
- TF/TFP unavailable;
- filterflow executable environment unavailable;
- implementation requires NumPy as BayesFilter algorithmic backend;
- matching filterflow requires mutating `.localsource/filterflow`;
- comparator fingerprint changes during a run;
- forbidden write/import is needed;
- unresolved Claude/Codex disagreement after max 5 rounds without human
  authorization.
- per-rung cells exceed the bounded runtime budget before reaching an
  interpretable pass/mismatch/blocker.

## Skeptical Pre-Execution Audit

- Wrong goal risk: do not veto on shared high residual.
- Wrong comparator risk: use local executable filterflow only, not pristine
  upstream and not student code.
- Scalar overclaim risk: gradient/surface rungs cannot prove gradient
  correctness unless scalar contract is reconciled.
- Smoothness overclaim risk: early 1D rungs do not validate the full 2D
  smoothness model.
- Hidden drift risk: record comparator fingerprints and preserve forbidden
  write sets.
- Artifact relevance: JSON/report must name first actual discrepancy or state
  none observed in executed bounded rungs.

Audit status: proceed to Claude review.

## Claude Review Protocol

Use Claude Code exactly as:

```bash
claude -p --model claude-opus-4-7 --effort max
```

Claude must return `ACCEPT` or `REJECT` with findings. Codex independently
classifies every finding as `ACCEPT`, `PARTIAL`, `DISPUTE`, or `CLARIFY`.
Patch accepted/partial findings and resubmit until `ACCEPT` or max 5 rounds.
If Codex disputes a finding, include the rebuttal in the next Claude prompt.
Use the same protocol for the result. On round 5, do not silently promote a
rejected plan/result; record inspection-only status unless the human explicitly
authorizes downstream use.
