## Required Review Use

Before any future implementation phase can pass, reviewers must:

1. add or update traceability rows for every new claim;
2. mark any missing anchor as `BLOCKED_UNTRACEABLE`;
3. mark any implemented but unvalidated source-backed feature as
   `BLOCKED_UNVALIDATED`;
4. record whether MATLAB is only an audit reference or a matched behavior
   target;
5. block public or scientific claims that outrun the row statuses.

## Current Blockers

- End-to-end fixed-branch score API is `BLOCKED_UNVALIDATED`.
- Stochastic-volatility benchmark reproduction remains a BayesFilter extension
  with scalar/tiny TT evidence only, not paper-scale reproduction.
- Spatial SIR benchmark is now a first-gate BayesFilter extension for
  model-contract evidence only; production TT/SIRT filtering, paper-scale
  `J=9` accuracy, and scalability remain unpromoted.
- Predator-prey benchmark is now a first-gate BayesFilter extension for
  model-contract and comparison-schema evidence only; matched
  linear/nonlinear preconditioning rows, usefulness evidence, paper-scale
  results, adaptive MATLAB behavior, and scalability remain unpromoted.
- Full adaptive Zhao--Cui derivative support is not claimed.
- MATLAB code cannot be copied into production BayesFilter code under the
  current clean-room policy.
