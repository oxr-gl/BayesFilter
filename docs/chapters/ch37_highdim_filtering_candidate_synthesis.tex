\chapter{Candidate Synthesis for Industrial-Scale Nonlinear Filtering}
\label{ch:bf-highdim-candidate-synthesis}

\section{Chapter Claim}
\label{sec:bf-highdim-synthesis-claim}

The most defensible industrial strategy is hybrid: use block-local Gaussian or
transport filters to stabilize high-dimensional filtering structure, then use
transport-preconditioned or learned-accelerated HMC only after target and
diagnostic gates pass.  This is a research ranking, not a default policy.

\section{Assumptions}
\label{sec:bf-highdim-synthesis-assumptions}

The ranking assumes the current BayesFilter evidence state: V1 nonlinear
sigma-point diagnostics are narrow, P8 rows are execution diagnostics only, no
NAWM nonlinear validation exists, and tensor/transport/HMC acceleration sources
still need deeper technical review before theorem-level claims.  The ranking is
therefore a low-confidence research-prior for planning evidence, not an
empirical method leaderboard.

\section{Ranking Criteria}
\label{sec:bf-highdim-ranking-criteria}

Candidates are ranked by expected scaling, required structure, degeneracy
behavior, differentiability, XLA/GPU readiness, implementation burden, evidence
burden, and suitability for a NAWM-style nonlinear DSGE target.

\section{Recommended Synthesis}
\label{sec:bf-highdim-synthesis-recommendation}

For NAWM-style nonlinear DSGE work, the recommended order is:
\begin{enumerate}[label=(\arabic*)]
  \item Build block-local deterministic and stochastic partitions.
  \item Establish IEKF/IEKS and block cubature references on small blocks.
  \item Add transport or flow proposals only with corrected weights.
  \item Use tensor or low-rank compression where rank diagnostics stay stable.
  \item Use fixed-trajectory HMC as the inspectable sampler baseline.
  \item Add NeuTra or HNN acceleration only after value/score and posterior
    diagnostics pass on the unaccelerated target.
\end{enumerate}

\section{BayesFilter Evidence Boundary}
\label{sec:bf-highdim-synthesis-evidence}

The V1 evidence supports current nonlinear value/score diagnostics and narrow
performance rows.  The P8 harness adds execution metadata for block extensions
only.  The ranking above is therefore a research roadmap.  It is not an
empirical winner table.

\section{Unresolved Claim Register}
\label{sec:bf-highdim-synthesis-unresolved}

\begin{itemize}
  \item No candidate is validated on NAWM.
  \item No candidate has a full high-dimensional posterior recovery benchmark.
  \item No GPU/XLA policy change is supported.
  \item No HMC acceleration method is promoted.
\end{itemize}

\section{What Is Not Concluded}
\label{sec:bf-highdim-synthesis-nonclaims}

The ranking does not conclude that the top method is scientifically correct or
that lower-ranked methods are poor.  It states a reviewer-defensible order for
building evidence under current BayesFilter constraints.
