\chapter{High-Dimensional Nonlinear Filtering Foundations}
\label{ch:bf-highdim-nonlinear-foundations}

\section{Chapter Claim}
\label{sec:bf-highdim-foundations-claim}

This chapter fixes notation and evidence boundaries for high-dimensional
nonlinear state-space filtering.  It does not certify any BayesFilter backend as
NAWM-ready, HMC-ready, or production-ready.  Its role is to make later
algorithmic chapters reviewable by separating exact filtering identities from
approximate filters, samplers, and performance diagnostics.

\section{Assumptions}
\label{sec:bf-highdim-foundations-assumptions}

The exact recursions in this chapter assume that the stated transition and
observation kernels exist on a common measurable state space, that the
predictive densities used in the likelihood factorization are finite and
positive on the observed path, and that all integrals are interpreted in the
measure induced by the model.  Approximate-filter statements assume only that a
backend declares the scalar value it computes; they do not assume that this
value is an accurate approximation to the exact likelihood.

\section{Nonlinear State-Space Contract}
\label{sec:bf-highdim-ssm-contract}

Let $\theta \in \Theta$ be a static parameter, $x_t \in \mathbb R^n$ a latent
state, and $y_t \in \mathbb R^m$ an observation.  A discrete-time nonlinear
state-space model has transition and observation kernels
\begin{align}
  x_t \mid x_{t-1},\theta &\sim f_\theta(\cdot \mid x_{t-1}),\\
  y_t \mid x_t,\theta &\sim g_\theta(\cdot \mid x_t).
\end{align}
Equivalently, when simulation maps exist,
\begin{align}
  x_t &= F_\theta(x_{t-1},\varepsilon_t), &
  y_t &= H_\theta(x_t,\eta_t),
\end{align}
with innovations $\varepsilon_t$ and observation shocks $\eta_t$.  The
BayesFilter implementation often separates stochastic integration coordinates
from deterministic completion coordinates.  In that case the declared
integration dimension, not the full state dimension alone, controls the number
of quadrature or particle coordinates that a backend must propagate.

\begin{definition}[Exact filtering recursion]
\label{def:bf-highdim-filtering-recursion}
Given an initial density $p_\theta(x_0)$, the prediction and update densities
are
\begin{align}
  p_\theta(x_t \mid y_{1:t-1})
    &= \int f_\theta(x_t \mid x_{t-1})
       p_\theta(x_{t-1}\mid y_{1:t-1})\,dx_{t-1}, \\
  p_\theta(x_t \mid y_{1:t})
    &= \frac{g_\theta(y_t\mid x_t)p_\theta(x_t\mid y_{1:t-1})}
            {p_\theta(y_t\mid y_{1:t-1})},
\end{align}
where
\begin{equation}
\label{eq:bf-highdim-predictive-density}
  p_\theta(y_t\mid y_{1:t-1})
    = \int g_\theta(y_t\mid x_t)p_\theta(x_t\mid y_{1:t-1})\,dx_t.
\end{equation}
\end{definition}

\begin{proposition}[Likelihood factorization]
\label{prop:bf-highdim-likelihood-factorization}
If the conditional predictive densities in
\eqref{eq:bf-highdim-predictive-density} exist and are finite, then
\begin{equation}
\label{eq:bf-highdim-loglik-factorization}
  \ell_T(\theta)
  =
  \log p_\theta(y_{1:T})
  =
  \sum_{t=1}^T \log p_\theta(y_t\mid y_{1:t-1}).
\end{equation}
\end{proposition}

\begin{proof}
Apply the probability chain rule
$p_\theta(y_{1:T})=\prod_{t=1}^T p_\theta(y_t\mid y_{1:t-1})$ and take logs
where the predictive terms are positive and finite.
\end{proof}

\section{Approximate Filtering Targets}
\label{sec:bf-highdim-approximate-targets}

Most high-dimensional nonlinear filters in this monograph replace the exact
prediction/update recursion by a controlled approximation.  A Gaussian
projection filter propagates moments rather than densities.  A particle filter
propagates an empirical measure.  A tensor-train filter propagates a compressed
density or operator.  A learned transport or HNN sampler changes coordinates or
proposal dynamics but still needs a target-preserving correction before exact
posterior language is allowed.

For a parameter prior $\pi(\theta)$, the exact posterior target is
\begin{equation}
\label{eq:bf-highdim-posterior-target}
  \log \pi(\theta\mid y_{1:T})
  =
  \ell_T(\theta) + \log \pi(\theta) - \log p(y_{1:T}).
\end{equation}
An approximate filter supplies $\widehat \ell_T(\theta)$ and therefore targets
the approximate posterior proportional to
$\exp\{\widehat \ell_T(\theta)\}\pi(\theta)$ unless a correction mechanism is
specified.

\section{Degeneracy In High Dimension}
\label{sec:bf-highdim-degeneracy}

High dimension changes which failures are ordinary.  Gaussian filters can lose
positive definiteness or collapse onto poor local linearizations.  Sigma-point
rules can grow from linear to exponential point counts.  Particle filters can
collapse to a small number of effective ancestors.  HMC can report finite values
but still move through a target geometry with divergences, poor energy
exploration, or unacceptable chain diagnostics.  Tensor compression can reduce
storage while introducing rank truncation, positivity, normalization, and
covariance-validity errors.

The NAWM-scale design target in Chapter~\ref{ch:bf-nawm-design-target} should be
read as a stress standard.  It motivates local structure, transport, analytic
gradients, and compiled execution, but it is not itself validation evidence.

\section{Evidence Ledgers}
\label{sec:bf-highdim-ledgers}

The chapters that follow use five ledgers:
\begin{enumerate}[label=(\roman*)]
  \item engineering correctness: code runs, shapes are stable, contracts are
    honored;
  \item numerical validity: finite values, stable covariance factors, parity
    against a stated comparator;
  \item sampler validity: HMC or Monte Carlo diagnostics satisfy a declared
    contract;
  \item scientific interpretation: the model and approximation answer the
    economic or statistical question;
  \item performance evidence: measured runtime, memory, compilation, and device
    behavior for exact tested cells.
\end{enumerate}
No ledger automatically promotes another.  A fast row with a failed validity
diagnostic is not a better method; it is a faster invalid or unresolved
diagnostic.

\begin{itemize}

  \item No exact nonlinear reference likelihood is established for BayesFilter

\section{What Is Not Concluded}
\label{sec:bf-highdim-foundations-nonclaims}

This chapter does not conclude production readiness, high-dimensional
tractability, GPU speedup, tensor-rank adequacy, HMC convergence, or NAWM
validation.  It provides the notation and claim discipline needed to test those
questions later.
